In this section an explanation about all the fields that you can find when creating a metric is provided.


Pattern:

This field determines the pattern of the metric.
A Percentage metric calculates the percentage of a set of instances that satisfy a condition over the total.
A Standard Deviation metric calculates the dispersion of an object over the global arithmetic mean.
A Frequency metric calculates the number of times that a value appears in the metric.

Type:

This field determines the type of the metric.
Evaluative type is used to detect a bad behaviour such as User Stories that don't follow the template.
Informative type is used for a better understanding of a repository. For instance, knowing the number of commits in a repository.

Subject:

This field determines the subject of the metric.
An individual metric is only applied to one student.
A team metric is applied to a whole team, which is made by some students.

Team Extension:

This field determines the team extension for an individual metric.
An overall team extension defines the associated global team metric.
A standard deviation team extension induce a team metric with a standard deviation category over the individual metric.

Object:

This field determines the domain object of the metric.
A task metric is defined for the tasks defined in the Taiga repository of the students.
A user story metric is defined for the user stories defined in the Taiga repository of the students.
A commit metric fot the commits defined in the GitHub repository of the students.

Modifier Type:

This field determines the modifier type of the metric.
A none modifier means that the metric is applied for all the instances, so there is not any restriction.
A state modifier is only applied for the boolean attributes of the tasks and user stories.
A sum modifier adds all the values of determined attributes.
A defined modifier applies for an attribute of the corresponding object.

Modifier Attribute:

This field determines the attribute in which the modifier apply in the metric.
A closed state applies for the tasks or user stories that are closed.
The sum of lines of code applies for the commits of a repository.
Estimated effort and actual effort are attributes for the defined type.

Condition Type:

This field determines the condition type of the metric.
A state condition is only applied for the boolean attributes of the tasks and user stories.
A defined type applies for an attribute of the corresponding object.
A card type gives the number of appearances of a multivalued attribute.
An apply type evaluates an external query.
A self type checks if the attribute value is equal to the member on which the metric is applied.
A Not Defined type applies for an attribute that doesn't belong to a metric.

Condition Attribute:

This field determines the attribute in which the condition apply in the metric.
A closed state applies for the tasks or user stories that are closed.
Estimated effort and actual effort are attributes for the defined type.
Committer attribute belongs to the student that has done the corresponding commit.
Responsible attribute belongs to the student that is responsible for a task.
Acceptance criteria belongs to user stories that has defined this attribute.
Check template US and check task reference are external queries for applying them in a metric.

Value:

This field determines the value for which the deviation is calculated in standard deviation metrics.
It can have the value of estimated effort - actual effort

Count Type:

This field determines the count type in frequency metrics.
A state type is only applied for the boolean attributes of the tasks and user stories.

Count Attribute:

This field determines the attribute in which the count apply in the metric.
A closed state applies for the tasks or user stories that are closed.

Metric name:

This field determines the name of the metric.

Description:

This field determines the description of the metric.
